EventSphere Platform Admin — Getting In

A plain-English walkthrough of the sign-in screens for our own staff console. No tech knowledge needed.

This document explains how EventSphere staff sign in to the Platform Admin console — the internal tool our team uses to run the whole business: approving organizer documents, releasing payouts, suspending accounts, and setting commission rates.

Think of the organizer sign-in as the front door of a shop — anyone can walk up and create an account. This console is the bank vault at the back. Different rules apply: nobody signs themselves up, everybody proves who they are twice, and every single action is recorded.

The one big difference to remember

There is no "Sign Up" button anywhere. You cannot create a platform admin account yourself. Someone already on the team must invite you. This is deliberate — it's the main thing protecting the vault.

The Big Picture — Two Ways In

Path 1 — Brand-new admin (first time ever):

Receives an emailed invite → Accepts invitation & sets password → Must set up two-factor security → Saves backup codes → Console

Path 2 — Existing admin (every normal day):

Log in with email + password → Enter 6-digit code from phone app → Console

Plus a few "something went wrong" screens: forgot password, session timed out, and access denied.

Path 2 · Everyday Sign-In

1 Log In

The first screen any staff member sees.

"I'm on the EventSphere ops team. I need to get into the console to review today's KYC queue."

What they see and can do

  • A clear header: "EventSphere Platform Admin — INTERNAL ACCESS ONLY." No marketing, no friendly welcome — it's obvious this isn't a customer page.
  • Type their work email and password. The eye icon shows/hides the password.
  • Press Continue.
  • Or use Sign in with corporate SSO (company single sign-on) instead.
  • Tap Forgot password? if needed.
  • A padlock notice warns: every sign-in attempt is recorded, including IP address and device.
  • The footer repeats: sessions are logged and audited · unauthorized access is prohibited.

What the screen looks like in different situations

SituationWhat the person sees
Normal readyClean form, waiting for details.
Wrong details error"Sign-in failed — your email or password is incorrect." Notice it never says which one was wrong — that's on purpose, so an outsider can't discover which emails exist.
Too many tries locked"Account temporarily locked — try again in 14 min 52 s, or contact your security administrator."
Account switched off error"Account deactivated — contact a platform super-admin to restore access." (Used when someone leaves the team.)
Unknown network blocked"You are connecting from an unrecognized network (203.0.113.44). This attempt has been logged." Signing in from an unexpected place gets stopped and recorded.

2 Two-Factor Check (the 6-digit code)

Appears straight after a correct password — every single time. There is no way to skip it.

"My password was right. Now the console asks for the 6-digit code from the authenticator app on my phone."

What they can do

  • Type the 6-digit authenticator code. The screen shows which account it's for (e.g. r.kulkarni@eventsphere.com).
  • Optionally tick "Trust this device for 7 days" — note it says clearly this is shorter than standard accounts, and they'll still be asked again on any new device or network.
  • Press Verify.
  • If their phone is lost: "Can't access your app? Use a backup code."

Situations

Normal readyCode box, ready to type.
Wrong code error"That code didn't match. Check your authenticator app and try again."
Code too old error"That code is no longer valid. Enter the current 6-digit code." (Codes refresh every 30 seconds.)
Backup-code modeSwaps to: "Enter one of the single-use backup codes you saved during setup," with a link back to the app.
Too many wrong codes locked"Verification locked. This challenge is locked for 15 minutes."

Path 1 · A Brand-New Admin Joins

3 Accept Your Admin Invitation

Where a new admin's journey starts — from a link in their email.

"Ravi added me to the platform team. I clicked the invite in my email and now I set up my account."

What they see and can do

  • Who invited them — e.g. "Invited by Ravi Kulkarni · Platform super-admin."
  • What powers they're getting — e.g. "ROLE · Platform operator," with tags like Tenant governance and KYC · payouts. So it's clear up front what they'll be able to do.
  • Type their full name and create a password. A strength meter and checklist appear: 12+ characters, upper & lower, a number, a symbol — stricter than the organizer side (which asks for 8+).
  • Press Accept & set up 2FA.
  • A note warns them plainly: they will be required to configure two-factor before reaching the console.

Situations

SituationWhat the person sees
Valid invite readyForm ready, showing inviter and role.
Expired warning"This invitation has expired — admin invitations are valid for 72 hours. Ask the super-admin who invited you to send a new one."
Revoked error"This invitation was revoked — access was withdrawn before you accepted."
Already used"Your account is already active. Sign in with the credentials you set up."

4 Set Up Two-Factor (Required, First Sign-In)

Immediately after accepting the invite. There is no skip button.

"Before I can see anything, I have to connect my authenticator app. The screen says it's mandatory."

What they can do

  • Read the banner: "Required · first sign-in" — "Two-factor is mandatory for every platform admin."
  • Scan a QR code with their authenticator app.
  • Or use the manual key shown on screen (e.g. JBSW Y3DP EHPK 3PXP GEZC) with a Copy button.
  • Helpful note: works with Google Authenticator, 1Password, Authy, or any similar app.
  • Type the 6-digit code to prove it worked, then press Confirm & continue.

Then — Save Your Backup Codes

  • A list of one-time codes appears (e.g. 4K2P-9QX7, 7M3D-1AZ8, B6VH-52NR…).
  • The screen is blunt: "Store these somewhere safe. Each code works once if you lose access to your authenticator. They won't be shown again."
  • They can copy or download them, and must tick to confirm they've saved them before continuing.

Situations

  • Scan & confirm normal — QR + key + code box.
  • Wrong code error — try again.
  • Backup codes — the one-time list, shown once only.
  • Loading — brief spinner.

Side-Doors · When Things Go Wrong

5 Forgot Password

From the "Forgot password?" link on the login screen.

"I can't remember my password. I'll ask for a reset link."

What they can do

  • Type their work email, press Send reset link, or go Back to sign in.

Situations

SituationWhat the person sees
Sent done"If an admin account exists for that address, we've sent password reset instructions. The link expires in 30 minutes." Note the careful wording — it never confirms whether the account is real. That's intentional security.
Resend cooldown"You can resend in 42s" — stops the button being spammed.

6 Reset Password

After clicking the reset link in the email.

"I clicked the link. Now I pick a new password."

What they can do

  • Type a new password (strength meter, strict rules) and confirm it. Press Update password.

Situations

SituationWhat the person sees
Mismatch error"Passwords don't match — both fields must be identical before you can continue."
Link expired error"For security, reset links expire after 30 minutes. Request a new one."
Success done"Password updated. Two-factor verification is still required at sign-in." — resetting a password does not bypass the 6-digit code.

7 Session Ended

Shown when someone gets signed out.

"I stepped away for a coffee. When I came back, the console had signed me out."

Two versions

SituationWhat the person sees
Idle timeout timed out"For security, this privileged session ended after 10 minutes of inactivity. Re-authenticate to continue." It even remembers where they were — e.g. "Return to System health after re-auth." (10 minutes is much shorter than the organizer dashboard, because this console can move money.)
Forced logout error"You were signed out — a platform super-admin ended your session remotely. If you believe this was a mistake, contact your security administrator." Used if someone needs to be cut off immediately.

8 Access Denied

Shown when a signed-in person reaches the console but isn't allowed in.

"I'm logged into EventSphere, but I'm not on the platform team — I landed on the admin console by mistake."

What they see and can do

  • "You're signed in, but your account doesn't have platform super-admin permissions. This console is restricted."
  • A warning: "This access attempt has been logged and reported to security."
  • Two ways out: Back to your dashboard or Sign in as a different user.

One-Line Summary of Every Screen

  • 1. Log In — staff sign in with work email + password.
  • 2. Two-Factor Check — the mandatory 6-digit code, every time.
  • 3. Accept Invitation — a new admin sets up their account from an invite.
  • 4. Two-Factor Setup — forced on first sign-in, ends with backup codes.
  • 5. Forgot Password — request a reset link.
  • 6. Reset Password — choose a new one (2FA still applies).
  • 7. Session Ended — timed out after 10 minutes, or cut off by another admin.
  • 8. Access Denied — you're signed in but not allowed here; it's been logged.
A note on devices. These screens work on phones, tablets, laptops and desktops, but they're built laptop-and-desktop first — this is an office tool our team uses at their desks, not something organizers open on the go.
